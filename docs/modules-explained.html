La carte du projet

Le but du projet : analyser un livre de Project Gutenberg (une bibliothèque gratuite en ligne) et en sortir une « fiche » (book card). Voici qui fait quoi, dans l'ordre où les fichiers s'appellent :

1bookworm.py  →  le chef d'orchestre : lit ta commande dans le terminal et lance la bonne analyse.
2gutenberg.py  →  le livreur : télécharge le texte du livre depuis Internet.
3cleaner.py  →  le nettoyeur : enlève les pages légales autour du vrai récit.
4utils/path_config.py  →  le secrétaire : sait où ranger les fichiers et fournit le texte propre.
5modules/cache.py  →  la mémoire : sauve/relit les résultats pour éviter de recalculer.
6modules/nlp.py  →  la boîte à outils langage : nettoie les mots, les compte, les transforme en chiffres.
7modules/lexdiv.py  →  une analyse : mesure la richesse du vocabulaire du livre.
★modules/topics.py  →  une analyse : trouve les grands thèmes (déjà documentée à part).

Règle d'or du projet : chaque analyse suit le même moule — une fonction run(book_id) qui regarde le cache, sinon récupère le texte, calcule, sauve, renvoie. Une fois que tu as compris lexdiv.py, tu les comprends tous.

Fichier 1 · le point de départ
🎼

bookworm.py

chef d'orchestre
Le fichier que tu lances dans le terminal. Il ne calcule rien lui-même : il décide quelle analyse appeler.
En clairTu tapes python bookworm.py --topics 11. Ce fichier lit l'option --topics et le numéro 11, appelle le bon module, puis affiche le résultat en JSON.
  • argparse est un outil Python qui lit les options de la ligne de commande (--lexdiv, --topics, etc.).
  • mutually_exclusive_group(required=True) = « tu dois choisir exactement une analyse, pas zéro, pas deux ».
  • run_task(args) regarde quelle option a été remplie et appelle le bon module.run(id).
  • Le résultat est transformé en texte JSON joli (indent=2) et affiché à l'écran.
  • Si une erreur survient, on l'affiche proprement et on renvoie le code 1 (signal d'échec).

L'import est fait à l'intérieur du if (ex. from modules import topics) : on ne charge un gros module que si on en a vraiment besoin. Démarrage plus rapide.

# Exemples de lancement dans le terminal :
python bookworm.py --lexdiv 11     # richesse du vocabulaire d'Alice
python bookworm.py --topics 11     # thèmes d'Alice
python bookworm.py --card 84       # fiche complète de Frankenstein
Fichier 2 · récupérer le texte
📦

modules/gutenberg.py

livreur
Télécharge le texte brut d'un livre depuis le site Project Gutenberg.
En clairOn donne un numéro de livre, la fonction essaie deux adresses Internet possibles, et renvoie le texte de la première qui marche.
  • requests est la librairie pour télécharger une page web depuis Python.
  • Gutenberg range parfois ses fichiers à deux endroits différents → on a une liste de 2 URLs à essayer dans l'ordre.
  • .format(id=book_id) remplace {id} par le vrai numéro dans l'adresse.
  • response.ok = vrai si le téléchargement a réussi → on renvoie le texte tout de suite.
  • Si aucune adresse ne marche, on lève une RuntimeError (« livre introuvable ») que bookworm.py attrapera.

timeout=30 : si le serveur ne répond pas en 30 secondes, on abandonne au lieu d'attendre indéfiniment.

Fichier 3 · enlever le superflu
🧹

modules/cleaner.py

nettoyeur
Un fichier Gutenberg contient des pages de licence avant et après le livre. Ce module ne garde que le récit.
En clairChaque livre Gutenberg a une ligne *** START OF … et une ligne *** END OF …. On coupe tout ce qui est avant la première et après la seconde.
  • text.find("*** START OF ") trouve la position du début du vrai livre (ou -1 si absent).
  • On garde le texte après cette ligne. Idem pour [Illustration] qu'on saute si présent.
  • text.find("*** END OF ") trouve la fin → on coupe tout ce qui suit.
  • .strip() enlève les espaces et lignes vides au début et à la fin.

Découper une chaîne avec text[debut:fin] s'appelle le slicing — un concept Python très courant qu'on retrouve partout dans le projet.

Fichier 4 · l'organisation des fichiers
🗂️

utils/path_config.py

secrétaire
Sait où sont rangés les fichiers, et fournit aux analyses un texte déjà téléchargé + nettoyé.
En clairPlutôt que d'écrire des chemins de fichiers partout, on les centralise ici. get_text(11) renvoie le récit d'Alice, prêt à analyser.
  • raw_path(id) → le chemin du texte brut, ex. data/raw/11.txt.
  • cache_path(id, "topics") → le chemin d'un résultat, ex. data/cache/11_topics.json.
  • get_text(id) : si le fichier existe déjà sur le disque, on le lit ; sinon on le télécharge (gutenberg) puis on le sauve pour la prochaine fois.
  • Avant de renvoyer, il passe le texte dans clean() → tu reçois toujours du texte propre.

Path (de la librairie pathlib) est la façon moderne de manipuler des chemins. RAW_DIR / "11.txt" assemble un chemin proprement, sans se soucier des / ou \ selon le système.

Fichier 5 · gagner du temps
💾

modules/cache.py

mémoire
Sauvegarde un résultat dans un fichier JSON et le relit plus tard — pour ne pas refaire un calcul long deux fois.
En clairCalculer les thèmes d'un livre prend du temps. Une fois fait, on l'écrit dans un fichier. La prochaine fois, on relit le fichier en une fraction de seconde.
  • save_json(path, data) : crée le dossier si besoin, puis écrit le dictionnaire en texte JSON lisible.
  • load_json(path, required_keys) : si le fichier n'existe pas → renvoie None (= « pas de cache, calcule »).
  • Si le fichier est abîmé ou n'a pas les bonnes clés attendues → on renvoie aussi None par sécurité.
  • required_keys.issubset(...) vérifie que toutes les clés attendues sont présentes (utile quand on change le format).

JSON est un format texte universel pour stocker des données (listes, dictionnaires, nombres). Python le lit/écrit avec le module json intégré.

Fichier 6 · la boîte à outils du langage
🔤

modules/nlp.py

outils texte
Le cœur « traitement du langage ». Les autres analyses piochent leurs outils ici. NLP = Natural Language Processing.
En clairTrois services : (a) simplifier les mots à leur forme de base, (b) transformer du texte en chiffres pour les maths, (c) gérer la liste des mots à ignorer.
  • lemmatize(text) utilise spaCy pour réduire chaque mot à sa racine : "running"→"run", "mice"→"mouse". Il garde uniquement noms, verbes, adjectifs, noms propres et jette les mots vides (the, of…).
  • vectorize() crée un TfidfVectorizer : il transforme du texte en tableau de chiffres en pondérant les mots importants (fréquents ici, rares ailleurs).
  • count_vectorize() : pareil mais compte juste combien de fois chaque mot apparaît (sans pondération).
  • _stop_words() construit la liste des mots à ignorer (« mots vides »).
Lemmatiser

Ramener un mot à sa forme de dictionnaire. Permet de compter run, runs et running comme un seul mot.

Stop words

Les mots trop courants (le, de, et…) qui n'apportent pas de sens et qu'on retire avant d'analyser.

Vectoriser

Transformer du texte en chiffres, car un ordinateur calcule sur des nombres, pas sur des lettres.

TF-IDF

Une note par mot : haute si le mot est fréquent dans ce texte mais rare dans les autres. = « ce qui rend ce texte unique ».

@functools.lru_cache(maxsize=1) sur _nlp() : charger spaCy est lent, donc on le garde en mémoire et on ne le charge qu'une seule fois, même si on l'appelle 100 fois.

Un livre peut être énorme : _spacy_chunks() le découpe en morceaux de 1 million de caractères car spaCy refuse les textes trop longs d'un coup.

modules/lexdiv.py

analyse
Mesure la diversité lexicale : un auteur utilise-t-il un vocabulaire riche et varié, ou répète-t-il souvent les mêmes mots ?
En clairOn compte les mots du livre de plusieurs façons et on en tire 6 indicateurs chiffrés. C'est l'exemple parfait du moule run() que suivent toutes les analyses.

Le squelette run(), identique partout dans le projet :

def run(book_id):
    path = cache_path(book_id, "lexdiv")
    cached = load_json(path, REQUIRED_KEYS)
    if cached is not None:      # déjà calculé ? on renvoie
        return cached
    text = get_text(book_id)         # sinon : récupérer le texte
    metrics = _compute_metrics(text) # calculer
    save_json(path, metrics)         # sauver pour la prochaine fois
    return metrics

Les 6 indicateurs calculés par _compute_metrics() :

CléNomCe que ça mesure
toktokensNombre total de mots dans le livre.
typtypesNombre de mots différents (le vocabulaire unique).
haphapaxMots qui n'apparaissent qu'une seule fois dans tout le livre.
ttrtype/token ratiotyp ÷ tok. Proche de 1 = très varié ; proche de 0 = répétitif.
mwlmean word lengthLongueur moyenne des mots (en lettres).
mwfmean word freqCombien de fois, en moyenne, chaque mot unique est utilisé.

Counter (de collections) est un compteur automatique : Counter(tokens) donne directement combien de fois chaque mot apparaît. Très pratique pour ce genre de stats.

tests
