Comment un livre entier se résume à cinq thèmes — et comment le code y arrive, étape par étape.

❦   ❦   ❦
En une phrase

Que fait ce fichier ?

Tu lui donnes un identifiant de livre (ex. Alice au pays des merveilles). Il télécharge le texte, le nettoie, puis en sort une liste de 5 grands thèmes, chacun décrit par ses mots les plus représentatifs — et il indique quel thème domine dans chaque partie du livre. Le résultat est mis en cache pour ne pas tout recalculer à chaque fois.

L'astuce centrale à comprendre

Un algorithme de « topic modeling » a besoin de plusieurs documents pour repérer des thèmes récurrents. Or ici, on n'a qu'un seul livre. La solution : découper le livre en tranches (~1500 mots chacune) et traiter chaque tranche comme un « document » distinct. Les thèmes émergent alors des mots qui reviennent ensemble d'une tranche à l'autre.

Le déroulé

Le pipeline, de run() au résultat

Six étapes seulement. Chaque étape transforme la donnée un peu plus : du texte brut jusqu'aux thèmes chiffrés.

1

run(book_id)  — le point d'entrée

topics.py

Regarde d'abord dans le cache. Si le résultat existe déjà → on le renvoie tel quel. Sinon, on lance tout le calcul puis on sauvegarde.

in : 11   →   out : { topics, sections }
2

get_text(book_id)  — récupérer le livre

utils/path_config.py

Lit le fichier local s'il existe, sinon le télécharge depuis Gutenberg. Puis clean() enlève l'en-tête/pied légal pour ne garder que le récit.

in : id   →   out : texte brut propre (str)
3

lemmatize(text)  — ne garder que le sens

nlp.py · spaCy

spaCy réduit chaque mot à sa forme de base et ne conserve que les noms, verbes, adjectifs et noms propres. "running"→"run", "mice"→"mouse". Les mots vides (the, of, and…) disparaissent.

in : texte   →   out : « alice rabbit hole fall curious … »
4

decouper_en_tranches(texte)  — couper en tranches

topics.py

Découpe les lemmes en tranches d'environ 1500 mots (au moins 4). C'est l'astuce centrale : créer plusieurs « documents » à partir d'un seul livre.

in : 1 long texte   →   out : [ "tranche 1", "tranche 2", … ]
5

trouver_themes(texte)  — faire émerger les thèmes

topics.py · le cœur

Cœur du module. Un seul TfidfVectorizer transforme les tranches en chiffres, puis la NMF décompose la matrice en 5 thèmes. Pour chaque thème → ses mots les plus forts (via mots_les_plus_forts()) ; pour chaque tranche → son thème dominant.

in : tranches   →   out : 5 topics + répartition par section
6

save_json(path, …)  — garder en mémoire

cache.py

Écrit le résultat sur disque. Au prochain appel, l'étape 1 le retrouvera et sautera tout le calcul.

in : dict résultat   →   out : fichier 11_topics.json

Les deux mots savants

TF-IDF & NMF, sans maths

le cœur

TF-IDF

« quels mots décrivent vraiment cette tranche ? »

Transforme du texte en nombres. Un mot reçoit un score élevé s'il est fréquent dans une tranche mais rare dans les autres. Résultat : chaque tranche devient une ligne de chiffres (un vecteur).

  • max_df=0.8 ignore un mot présent dans +80 % des tranches (ex. « alice »).
  • « alice » est partout → score faible, peu distinctif.
le cœur

NMF

« regroupe ces mots en quelques thèmes »

Prend le grand tableau de chiffres et le factorise en 5 paquets de mots qui apparaissent souvent ensemble. Chaque paquet = un thème. Comme tous les nombres restent positifs, les thèmes sont lisibles (additions de mots, jamais de soustractions).

  • random_state=42 → résultat reproductible.
  • Le label d'un thème = ses 3 mots les plus forts.

C'est fait

Ce qui a été simplifié

Le code ci-dessous n'est plus une proposition : c'est le vrai contenu de topics.py aujourd'hui. La version « pro » d'origine a été allégée — moins de magie, plus de lisibilité — tout en gardant ce qui assure la stabilité (cache, résultat reproductible, garde-fous anti-crash).

◆ Avant (niveau avancé)

  • from __future__ import annotations, types dict[str, Any], set[str] partout.
  • Manipulation directe des matrices sklearn, répétée à plusieurs endroits.
  • Vocabulaire candidat + double vectorisation — subtil à suivre.
  • Découpe de sections avec gestion du reste (remainder, fusion de la dernière).
  • 8 constantes de réglage à comprendre d'un coup.

◆ Maintenant (en place)

  • Aucun typage avancé : juste des fonctions claires, nommées en français.
  • Un helper mots_les_plus_forts() au lieu de trier les matrices partout.
  • Étape du vocabulaire candidat supprimée : un seul TfidfVectorizer.
  • Découpe simple : tranches de taille régulière, sans cas particulier.
  • 6 constantes, chacune commentée sur sa ligne.
Le gros levier appliqué : retirer _candidate_vocabulary. Elle améliorait un peu la qualité, mais c'était la partie la plus difficile à expliquer. Sans elle, le pipeline tombe à 4 étapes simples : nettoyer → découper → vectoriser → NMF — et les thèmes restent très bons (testé sur Alice).
1
Découpe simple

Une boucle range(0, len, taille) sans gérer le reste. 13 lignes au lieu de 16, et zéro cas particulier.

2
Un seul vectoriseur

vectorize(...) appelé une seule fois dans trouver_themes(), au lieu de deux vectorisations.

3
Stabilité gardée

Cache, random_state=42, garde-fou min(...) et conversions float() conservés exprès.

Le code réel, en place — ~115 lignes au lieu de 145, lisible de haut en bas :

from math import ceil
from sklearn.decomposition import NMF
from modules.cache import load_json, save_json
from modules.nlp import lemmatize, vectorize
from utils.path_config import cache_path, get_text

NB_THEMES = 5          # combien de themes on veut trouver
MOTS_PAR_THEME = 10     # mots gardes pour decrire chaque theme
TAILLE_TRANCHE = 1500   # taille visee d'une tranche, en mots
MIN_TRANCHES = 4        # au moins 4 tranches pour la NMF
MAX_RATIO = 0.8          # on ignore un mot present dans +80% des tranches

CACHE_VERSION = 5
REQUIRED_KEYS = {"version", "method", "topics", "sections"}


def decouper_en_tranches(texte):
    mots = texte.split()
    if not mots:
        return []

    nb_tranches = max(MIN_TRANCHES, len(mots) // TAILLE_TRANCHE)
    taille = ceil(len(mots) / nb_tranches)

    tranches = []
    for debut in range(0, len(mots), taille):
        tranches.append(" ".join(mots[debut:debut + taille]))
    return tranches


def mots_les_plus_forts(poids, mots):
    total = poids.sum()
    # argsort() trie du + petit au + grand ; [::-1] inverse
    positions_triees = poids.argsort()[::-1]

    resultat = []
    for position in positions_triees[:MOTS_PAR_THEME]:
        if poids[position] <= 0:
            continue
        score = float(poids[position] / total) if total else 0.0
        resultat.append({"word": mots[position], "score": score})
    return resultat


def trouver_themes(texte):
    tranches = decouper_en_tranches(texte)
    if not tranches:
        return resultat_vide()

    # 1) texte -> chiffres (TF-IDF), un seul vectoriseur
    vectoriseur = vectorize(stop_words="english", max_df=MAX_RATIO)
    matrice = vectoriseur.fit_transform(tranches)
    mots = vectoriseur.get_feature_names_out()

    # 2) NMF : on ne demande pas + de themes que de tranches/mots
    nb_themes = min(NB_THEMES, matrice.shape[0], matrice.shape[1])
    modele = NMF(n_components=nb_themes, init="nndsvda",
                 random_state=42, max_iter=600)
    poids_tranches = modele.fit_transform(matrice)

    # 3) mots forts par theme   4) theme dominant par tranche
    themes = []
    for numero, poids in enumerate(modele.components_):
        mots_du_theme = mots_les_plus_forts(poids, mots)
        label = " / ".join(m["word"] for m in mots_du_theme[:3])
        themes.append({"topic": numero + 1, "label": label,
                       "words": mots_du_theme})

    sections = []
    for numero, poids in enumerate(poids_tranches):
        total = poids.sum()
        repartition = [{"topic": i + 1,
                        "score": float(v / total) if total else 0.0}
                       for i, v in enumerate(poids)]
        sections.append({
            "section": numero + 1,
            "token_count": len(tranches[numero].split()),
            "dominant_topic": int(poids.argmax()) + 1 if len(poids) else 1,
            "topic_distribution": repartition,
        })

    return {"version": CACHE_VERSION, "method": "nmf_tfidf",
            "topics": themes, "sections": sections}


def run(book_id):
    path = cache_path(book_id, "topics")

    cached = load_json(path, REQUIRED_KEYS)   # deja calcule ?
    if cached is not None:
        return cached

    texte = get_text(book_id)                 # sinon on calcule
    resultat = trouver_themes(lemmatize(texte))
    save_json(path, resultat)                 # et on sauve
    return resultat
